# Règles de Planning — CS Consulting Stratégique

## Préférences de Catherine

### Horaires de travail
- **Début de journée** : 9h00
- **Fin de journée** : 17h00 (flexible jusqu'à 18h si nécessaire)
- **Pause déjeuner** : 12h00-13h00 (minimum, peut aller jusqu'à 13h30)
- **Jours travaillés** : Lundi → Vendredi
- **Week-end** : PAS de planification sauf demande explicite

### Principes de time blocking

#### Blocs du matin (9h-12h) — Travail profond
- Tâches créatives : création de contenu, rédaction de guides
- Tâches stratégiques : feuilles de route, préparation de sessions
- Tâches qui demandent de la concentration
- ⚠️ Éviter les petites tâches admin le matin

#### Blocs de l'après-midi (13h/14h-17h) — Tâches opérationnelles
- Sessions clients (souvent l'après-midi)
- Admin : emails, factures, relances
- Prospection, CRM
- Petites tâches regroupées en un seul bloc

#### Blocs buffer
- **15 min minimum** entre deux créneaux différents
- **30 min** avant une session client (temps de préparation mentale)
- **30 min** après une session client (notes rapides)

### Capacité journalière
- **Maximum 6h de travail planifié** sur 8h de journée
- Les 2h restantes = marge pour :
  - Imprévus
  - Emails qui arrivent
  - Pauses
  - Transitions entre tâches

### Durée type par activité

| Activité | Durée recommandée |

| Admin & emails | 30 min - 1h (1 seul bloc) |

## Format des événements Google Calendar

### Convention de nommage

```
[Emoji] Titre clair et concis
```

**Exemples :**
- 🎯 Préparer session Fred — feuille de route
- 💰 Factures : relance Client X + facture OPCO
- 🎬 Création vidéo LinkedIn — sujet à définir
- 📥 Trier Brain Dump
- 📊 Revue de semaine
- ☀️ Briefing du matin
- 🟡 Admin : emails + petites tâches
- 📧 Prospection : relancer 3 contacts CRM

### Description de l'événement

Inclure dans la description :
1. **Objectif** — ce qui doit être accompli à la fin du créneau
2. **Contexte** — pourquoi c'est important / pour quel client
3. **Lien Notion** — si la tâche est liée à une page Notion
4. **Checklist** — si plusieurs sous-tâches à faire pendant le créneau

**Exemple :**
```
Objectif : Finaliser la feuille de route de Fred pour la session de jeudi
Contexte : Session #5, on aborde la restructuration de son offre
Notion : https://notion.so/2efc3a2f425581c08c3afd9a5b39c6c0
☐ Relire les notes de la session #4
☐ Préparer 3 options de positionnement
☐ Mettre à jour le dashboard client
```

## Interactions avec les autres skills

### weekly-planner → agenda-writer
Le flux idéal :
1. Catherine lance le `weekly-planner` → obtient ses priorités de la semaine
2. Catherine dit "planifie ça dans mon agenda"
3. `agenda-writer` prend les priorités → propose un planning → écrit dans Calendar

### Tâches client (session-report, client-onboarding)
- Si une session est prévue dans la semaine → bloquer automatiquement un créneau de préparation (45 min, la veille ou le matin même)
- Si un onboarding est en cours → bloquer les créneaux pour chaque étape (dashboard, kit de bienvenue, etc.)

### Facturation (invoice-generator)
- Si des échéances arrivent → bloquer 30 min pour émettre et envoyer les factures
- Si des relances sont nécessaires → les intégrer dans le bloc Admin
