⚙️ Workflows & Processus Clés

Workflow Client Standard

De la Découverte à la Production

Étapes:

  1. Questionnaire de découverte (Questionnaire Onboarding)
  2. Audit & diagnostic initial
  3. Proposition commerciale (Proposal Generator)
  4. Préparation session kickoff (Agenda Writer)
  5. Sessions de travail / implementation
  6. Rapports de session (Session Report)
  7. Suivi & optimisation continue

Workflow Projets de Développement

Conception à Déploiement

  1. Briefing & requirements
  2. Architecture planning (Plan Agent)
  3. Développement (VS Code + Claude Code)
  4. Testing & QA
  5. Security review
  6. Déploiement (Vercel/GitHub Pages)
  7. Monitoring & maintenance

Workflow de Collaboration Interne

Memory & Async Collaboration

  • Auto-memory persistante (next session context)
  • CLAUDE.md pour documentation projet
  • TodoWrite pour tracking tâches
  • Scheduled agents pour tasks récurrentes
  • Git commits avec contexte clair

Data Flow & Intégrations

L'architecture connectée :

  • Clients → Questionnaire Onboarding → Proposal Generator
  • Sessions → Notes → Session Report Generator
  • Billing → Invoice Generator → Stripe
  • Communication → Gmail/Telegram → Automations (N8N)
  • Data → Supabase PostgreSQL → Edge Functions/APIs
  • Frontend → Portail Client V2 → Backend APIs
  • Deployments → GitHub → Vercel/GitHub Pages

📌 Recommandations d'Intégration pour Mickaël

🎯 Priorités Immédiates :

Phase 1: Onboarding (Week 1)

  1. Explorer ce guide & tous les repos
  2. Prendre connaissance du système de memory
  3. Parcourir les skills & leur documentation
  4. Vérifier accès à tous les outils (GitHub, Supabase, etc.)
  5. Créer sa propre configuration locale CLAUDE.md

Phase 2: Quick Wins (Week 2-3)

  1. Contribuer sur un petit projet de dev (bug fix ou feature)
  2. Assister une session client (Fred ou FSY)
  3. Générer un premier rapport avec /session-report
  4. Créer sa première proposition avec /proposal-generator

Phase 3: Ownership (Week 4+)

  1. Prendre en charge un client ou projet
  2. Optimiser workflows existants
  3. Créer de nouveaux skills si besoins
  4. Documenter découvertes & bonnes pratiques
