KEY DESIGN DECISIONS
================================================================================

1. Async Processing via Bull Queue
   - Rationale: Meet 1-minute SLA for 100 invoices
   - Implementation: 5 concurrent workers, exponential retry
   - Trade-off: Eventual consistency (mitigated by 202 Accepted + polling)

2. JWT Authentication with RBAC
   - Rationale: Stateless, scalable, standard approach
   - Implementation: 15m access token + 30d refresh token + 3 roles
   - Security: Token verification, expiration checking, HTTPS-only

3. Immutable Audit Log
   - Rationale: Tax compliance (7-year retention)
   - Implementation: Append-only table, captures old/new values
   - Benefit: Can audit "who did what when" for any invoice

4. PostgreSQL with ACID Transactions
   - Rationale: Financial data consistency (no loss, no duplicates)
   - Implementation: FK constraints, check constraints, transactions
   - Benefit: Database prevents invalid states
