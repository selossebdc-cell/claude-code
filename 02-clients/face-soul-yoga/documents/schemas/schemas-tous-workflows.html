💳 Workflow 01 — Stripe Achat → Brevo Active
⚠️ C'est LE workflow le plus important. Si celui-ci ne marche pas, rien ne marche après l'achat.
1
Déclenché par
🔔 Webhook Stripe : charge.succeeded
⬇️
2
n8n reçoit
Email du client + montant + product_id (FSY ou MTM)
{ "event": "charge.succeeded", "customer_email": "john@example.com", "amount": "1700" (17€ en cents), "product_id": "fsy_studio OR mtm", "charge_id": "ch_1ABC123..." }
⬇️
3
n8n fait un choix
Si product_id = fsy_studio, alors OFFRE=fsy_studio. Si product_id = mtm, alors OFFRE=mtm
⬇️
4
n8n met à jour Brevo
Crée ou met à jour le contact avec ces attributs :
Email: john@example.com STATUT_MEMBRE: active ✓ OFFRE: fsy_studio (ou mtm) ✓ DATE_ACHAT: 2026-04-20 ✓ SOURCE: stripe (ou conserve youtube si existant)
✅ Après cette étape : Brevo détecte que STATUT_MEMBRE=active. L'automation Brevo "Onboarding FSY Studio" (ou MTM) s'active automatiquement.
🔧 Configuration Détaillée
Config Valeur À vérifier
Webhook URL (Stripe) https://n8n.srv921609.hstgr.cloud/webhook/stripe-fsy-01 Est-ce que Stripe envoie ici ? (Stripe Dashboard → Webhooks → Event Deliveries)
Brevo Credentials API Key configurée dans n8n Credential valide ? (n8n → Credentials → Brevo)
Attributs Brevo STATUT_MEMBRE, OFFRE, DATE_ACHAT existent Brevo → Contacts → Attributes (vérifier qu'ils existent)

1. Vérifier Stripe Webhooks Event Deliveries (webhook reçu ?)
2. Vérifier n8n logs (Workflow 01 a-t-il été exécuté ?)
3. Vérifier Brevo API credentials dans n8n

📧 Workflow 09 — Pré-Annulation J-7
Rétention : 7 jours avant le renouvellement, envoyer un email de rétention pour éviter l'annulation.
1
Déclenché par
⏰ Schedule : Quotidien (vérifie les dates J-7)
⬇️
2
n8n calcule
Pour chaque client : DATE_ACHAT + 30j = date renouvellement. Si DATE_RENOUVELLEMENT = aujourd'hui + 7j, c'est un candidat.
Cherche dans Brevo: - DATE_ACHAT: 2026-03-20 - Renouvellement: 2026-04-20 - Aujourd'hui: 2026-04-13 - Différence: 7 jours → ALERTE!
⬇️
3
Ajouter attribut
ALERTE_RENOUVELLEMENT = true
⬇️
4
Brevo automation se déclenche
Email rétention : "On t'aime, continue !" / "Tu as oublié quelque chose ?"
✅ Résultat : Avant l'annulation, le client reçoit un dernier email pour le retenir.
