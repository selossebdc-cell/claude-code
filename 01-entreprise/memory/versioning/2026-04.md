### 2026-04-27 | BUG FIX | Portail Client V2 — Template RLS isolation bug
- **What**: Fixed critical isolation bug where Taïna (Guadeloupe Explor) could see Face Soul Yoga templates
- **Root Cause**: Admin client selector loaded profiles without `client_id` column, assigned user ID instead of client_id to `appState.currentClientId`
- **Fix**: 
  - Added `client_id` to SELECT query (line 1337)
  - Changed `opt.value = c.id` to `opt.value = c.client_id` (line 1343)
- **Testing**: ✅ Verified — Taïna sees 0 templates, Aurélia sees 13 templates
- **Commits**: ae800bc (code fix), ae9157d (documentation)
- **Memory Created**: 
  - `project_portail-client-v2-template-isolation.md` (issue + solution + architecture)
  - `feedback_rls-testing-methodology.md` (debugging approach for RLS isolation bugs)
- **Impact**: Multi-tenant isolation now enforced correctly at both DB (RLS) and frontend (appState) layers
  - Structured proposition output format
  - Quality checklist before sending report
- **Impact**: Weekly reports will be high-quality, actionable, non-overlapping

  - `factory-spec` — MODEL phase (technical specifications)

- **New Document**: `/technical/project-factory-status.md` — Real-time tracking of factory adoption per project
